# Règle : Pas de fichiers Markdown inutiles

## Interdit
- Créer des fichiers .md de documentation (GUIDE.md, EXAMPLES.md, PROMPTS.md, etc.)
- Créer des fichiers de résumé ou récapitulatif
- Documenter le travail accompli dans des fichiers séparés

## Autorisé
- README.md (un seul, concis)
- Fichiers .md explicitement demandés par l'utilisateur

## Principe
Le code parle de lui-même. Pas besoin de 10 fichiers markdown pour expliquer comment utiliser 3 endpoints.
